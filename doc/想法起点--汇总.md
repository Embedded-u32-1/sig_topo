## 三、核心概念（MVP 精简版）
| 概念 | 定义 |
|---|---|
| 拓扑（Topology） | 一份描述文件对应的完整逻辑集合，包含所有信号与转移规则 |
| 信号（Signal） | 独立的状态载体，对应业务中一个状态变量，全局唯一ID |
| 状态（State） | 信号的合法取值，字符串枚举型，一个信号同一时间只有一个状态 |
| 事件（Event） | 触发状态转移的外部输入，由业务代码主动投递 |
| 转移（Transition） | 信号在指定事件下，从源状态切换到目标状态的规则 |
| 动作（Action） | 状态变化时触发的业务副作用；描述文件仅声明动作ID，具体逻辑由业务代码注册实现 |

## 四、JSON 描述规范
### 4.1 顶层 Schema
```json
{
  "version": "0.1",
  "signals": [ /* 信号定义数组 */ ],
  "transitions": [ /* 转移规则数组 */ ]
}
```

### 4.2 信号定义（signals）
每个信号为一个对象，字段说明：
- `id`：string，必填，信号唯一标识，全局不重复
- `initial_state`：string，必填，初始状态，必须属于 `states` 列表
- `states`：array<string>，必填，合法状态集合，无重复值

示例片段：
```json
"signals": [
  {
    "id": "task_status",
    "initial_state": "idle",
    "states": ["idle", "running", "success", "failed"]
  }
]
```

### 4.3 转移规则定义（transitions）
每个转移为一个对象，字段说明：
- `signal_id`：string，必填，所属信号ID，必须在 `signals` 中存在
- `from`：string，必填，源状态；取值为具体状态名，或 `*` 表示匹配任意状态
- `event`：string，必填，触发事件名
- `to`：string，必填，目标状态，必须属于该信号的合法状态
- `actions`：object，可选，分生命周期绑定动作ID
  - `on_exit`：array<string>，退出源状态前执行的动作，按数组顺序执行
  - `on_transition`：array<string>，状态切换完成后、进入目标状态前执行
  - `on_enter`：array<string>，进入目标状态后执行

**执行顺序**：`on_exit` → 状态更新 → `on_transition` → `on_enter`

#### 引擎主结构体与方法
```rust
type ActionFn = Box<dyn FnMut(ActionContext) -> Result<(), EngineError>>;

pub struct TopologyEngine {
    schema: TopologySchema,
    signal_states: HashMap<String, String>,
    actions: HashMap<String, ActionFn>,
}

impl TopologyEngine {
    /// 从 JSON 字符串加载拓扑并初始化引擎
    pub fn from_json(json_str: &str) -> Result<Self, EngineError> {
        let schema: TopologySchema = serde_json::from_str(json_str)
            .map_err(|e| EngineError::ParseError(e.to_string()))?;
        
        // 静态校验
        Self::validate(&schema)?;

        // 初始化所有信号为初始状态
        let mut signal_states = HashMap::new();
        for signal in &schema.signals {
            signal_states.insert(signal.id.clone(), signal.initial_state.clone());
        }

        Ok(Self {
            schema,
            signal_states,
            actions: HashMap::new(),
        })
    }

    /// 注册业务动作实现
    pub fn register_action<F>(&mut self, action_id: &str, f: F)
    where
        F: FnMut(ActionContext) -> Result<(), EngineError> + 'static,
    {
        self.actions.insert(action_id.to_string(), Box::new(f));
    }

    /// 向指定信号投递事件，触发状态转移
    pub fn send_event(
        &mut self,
        signal_id: &str,
        event: &str,
        payload: Option<Value>,
    ) -> Result<TransitionResult, EngineError> {
        // 1. 校验信号存在
        let current_state = self
            .signal_states
            .get(signal_id)
            .ok_or_else(|| EngineError::SignalNotFound(signal_id.to_string()))?
            .clone();

        // 2. 匹配转移规则
        let transition = self
            .schema
            .transitions
            .iter()
            .find(|t| {
                t.signal_id == signal_id
                    && (t.from == current_state || t.from == "*")
                    && t.event == event
            })
            .ok_or_else(|| EngineError::TransitionNotFound {
                signal: signal_id.to_string(),
                event: event.to_string(),
            })?;

        let target_state = &transition.to;
        let mut executed = Vec::new();

        // 3. 构建上下文
        let ctx = ActionContext {
            signal_id: signal_id.to_string(),
            from_state: current_state.clone(),
            to_state: target_state.clone(),
            event: event.to_string(),
            payload,
        };

        // 4. 按生命周期执行动作
        Self::run_actions(&mut self.actions, &transition.actions.on_exit, &ctx, &mut executed)?;

        // 5. 更新状态
        self.signal_states
            .insert(signal_id.to_string(), target_state.clone());

        Self::run_actions(&mut self.actions, &transition.actions.on_transition, &ctx, &mut executed)?;
        Self::run_actions(&mut self.actions, &transition.actions.on_enter, &ctx, &mut executed)?;

        Ok(TransitionResult {
            signal_id: signal_id.to_string(),
            from: current_state,
            to: target_state.clone(),
            executed_actions: executed,
        })
    }

    /// 查询信号当前状态
    pub fn get_state(&self, signal_id: &str) -> Result<&str, EngineError> {
        self.signal_states
            .get(signal_id)
            .map(|s| s.as_str())
            .ok_or_else(|| EngineError::SignalNotFound(signal_id.to_string()))
    }

    // --- 内部工具方法 ---

    fn validate(schema: &TopologySchema) -> Result<(), EngineError> {
        // 校验信号ID不重复
        let mut signal_ids = std::collections::HashSet::new();
        for signal in &schema.signals {
            if !signal_ids.insert(&signal.id) {
                return Err(EngineError::ValidationError(format!(
                    "Duplicate signal id: {}",
                    signal.id
                )));
            }
            // 校验初始状态合法
            if !signal.states.contains(&signal.initial_state) {
                return Err(EngineError::ValidationError(format!(
                    "Initial state '{}' not in signal '{}' states",
                    signal.initial_state, signal.id
                )));
            }
        }

        // 校验转移引用的信号、状态均存在
        for trans in &schema.transitions {
            let signal = schema
                .signals
                .iter()
                .find(|s| s.id == trans.signal_id)
                .ok_or_else(|| {
                    EngineError::ValidationError(format!(
                        "Transition references unknown signal: {}",
                        trans.signal_id
                    ))
                })?;

            if trans.from != "*" && !signal.states.contains(&trans.from) {
                return Err(EngineError::ValidationError(format!(
                    "Transition from state '{}' not in signal '{}'",
                    trans.from, trans.signal_id
                )));
            }
            if !signal.states.contains(&trans.to) {
                return Err(EngineError::ValidationError(format!(
                    "Transition to state '{}' not in signal '{}'",
                    trans.to, trans.signal_id
                )));
            }
        }

        Ok(())
    }

    fn run_actions(
        actions: &mut HashMap<String, ActionFn>,
        action_ids: &[String],
        ctx: &ActionContext,
        executed: &mut Vec<String>,
    ) -> Result<(), EngineError> {
        for id in action_ids {
            let f = actions
                .get_mut(id)
                .ok_or_else(|| EngineError::ActionNotFound(id.clone()))?;
            f(ctx.clone())?;
            executed.push(id.clone());
        }
        Ok(())
    }
}
```

## 六、标准执行流程
1. **加载阶段**：读取 JSON 文件 → 反序列化为 Schema → 执行静态校验 → 初始化所有信号为初始状态
2. **注册阶段**：业务代码调用 `register_action`，将动作ID与具体实现函数绑定
3. **运行阶段**：业务代码调用 `send_event` 投递事件
   1. 校验信号存在，获取当前状态
   2. 遍历转移规则，匹配 `from` 与 `event`
   3. 按顺序执行 `on_exit` 动作
   4. 更新信号状态为目标状态
   5. 按顺序执行 `on_transition`、`on_enter` 动作
   6. 返回转移结果

## 七、业务接入示例
```rust
use signal_topology::{TopologyEngine, ActionContext, EngineError};

fn main() -> Result<(), EngineError> {
    // 1. 加载拓扑
    let json = include_str!("topology.json");
    let mut engine = TopologyEngine::from_json(json)?;

    // 2. 注册业务动作
    engine.register_action("log_idle_leave", |ctx: ActionContext| {
        println!("离开 idle 状态，事件：{}", ctx.event);
        Ok(())
    });

    engine.register_action("init_task_resource", |_| {
        println!("初始化任务资源");
        Ok(())
    });

    engine.register_action("start_task_execution", |_| {
        println!("开始执行任务");
        Ok(())
    });

    engine.register_action("notify_task_success", |_| {
        println!("任务成功通知");
        Ok(())
    });

    // 3. 投递事件，触发流转
    println!("初始状态：{}", engine.get_state("task_status")?);

    let result = engine.send_event("task_status", "start", None)?;
    println!("转移后状态：{}", engine.get_state("task_status")?);
    println!("执行的动作：{:?}", result.executed_actions);

    let result = engine.send_event("task_status", "finish", None)?;
    println!("最终状态：{}", engine.get_state("task_status")?);

    Ok(())
}
```

## 八、MVP 验收标准
1. 合法 JSON 文件加载成功，所有信号初始状态正确
2. 投递匹配事件，状态按规则切换，绑定动作按顺序执行
3. 投递不匹配事件，状态不变化，返回明确错误
4. 非法拓扑（重复信号、不存在的状态）在加载阶段即校验失败
5. 修改 JSON 中的转移规则或动作绑定，业务代码零修改，重新加载后逻辑立即生效
